Virtualisasi dengan KVM di Debian SID GNU/Linux part 1

Mon 30 January 2017 • Virtualisasi •

Instalasi qemu/kvm di debian linux, virtualisasi dengan linux kvm

KVM adalah singkatan dari Kernel-based Virtual Machine, merupakan salah satu arsitektur virtualisasi bersumber terbuka (Open source) yang ditanamkan kedalam linux kernel. KVM merupakan virtualisasi penuh yang memanfaatkan extensi virtualisasi pada prosesor X86. Untuk menjalankan module KVM pada distribusi linux kita cukup mengenable extensi virtualisasi dari BIOS dan meload kernel modulenya kvm.ko atau jika kita hanya ingin module spesifik dengan vendor hardware yang kita gunakan yang di load maka untuk intel kvm-intel.ko dan untuk AMD kvm-amd.ko.

Alasan saya sendiri lebih memilih qemu/kvm dibanding virtualbox atau vmware adalah selain open source (well saya tidak panatik dengan FOSS sebenarnya :sweat_smile: ) virtualbox dan vmware belakangan semakin sering bermasalah di system saya ketika habis melakukan upgrade system.

Instalasi

Untuk menginstall KVM dan paket yang didukungnya di Debian SID maka kita lakukan dengan cara berikut:

$ sudo aptitude install qemu-kvm libvirt-daemon-system libvirt-clients libvirt-dev

Secara default KVM hanya mengijinkan root atau superuser yang dapat menjalankan dan mengadministrasi virtual machine, untuk dapat menjalankan dan mengatur virtual machine pada kvm dengan user biasa maka tambahakan user yang kita gunakan groups libvirt dan kvm.

$ sudo usesmod -aG kvm user
$ sudo usermod -aG libvirt user

Untuk manajemen dan administrasi kvm memiliki beberapa opsi, untuk cli di terminal dengan menggunakan virtinst atau menggunakan virsh, sedangkan untuk graphical tools kita bisa menggunakan virtmanager atau gnome boxes.

Penggunaan

Virtinst

Tools ini berfungsi untuk mempermudah pembuatan VM, selain kvm tools ini juga dapat digunakan untuk XEN, dan linux container seperti lxc contoh penggunaan:

virt-install --virt-type kvm --name archvm \
--connect qemu:///system --network bridge:br0 \
--arch=x86_64  --os-type=linux  \
--location http://kambing.ui.ac.id/iso/archlinux/2017.03.01/archlinux-2017.03.01-dual.iso/ \
--graphic vnc --disk path=/var/lib/libvirt/images/test.img,size=4 \
--memory 500 --vcpus=2

Perintah ini akan membuat VM baru dengan guest arch linux, dengan arsitektur x86_64, menggunakan br0 sebagai network interface, dengan 2 virtual cpu, 500mb ram dan membuat virtual disk dengan size 4gb, vnc sebagai graphic serta location installer di mirror kambing.ui.ac.id

Untuk opsi lebih lengkap tentang virt-install bisa dilihat dengan perintah : man virt-install atau virt-install --help atau membacanya di: sini

Virsh

Virsh adalah CLI tools untuk membuat, menghapus, mengedit dan menghentikan virtual machine contoh penggunaan:

  • Untuk terhubung ke VM
$ virsh connect qemu:///system
  • melihat daftar atau list VM yang berjalan
$ virsh list

Virsh command juga dapat digunakan untuk membuat VM menggunakan virsh command maka, kita harus membuat file konfig untuk VM kita dalam format XML, untuk sample libvirt XML bisa di merujuk di sini,

  • Untuk Membuat VM baru dari xml file dan menjalankannya secara otomatis
$ virsh create /tmp/vmbaru.xml
  • Mendefinisikan VM baru dari file XML tanpa menjalankannya secara otomatis
$ virsh define /tmp/foo_new.xml
  • Menjalankan, menghentikan dan reboot VM dengan virsh:
$ virsh start foo

$ virsh reboot foo

$ virsh shutdown foo

$ virsh suspend foo

$ virsh resume foo

Untuk perintah lainya bisa dicek di man virsh
